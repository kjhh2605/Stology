# Stology 데모사이트 작업 지침

이 파일은 `demo/` 아래 React/Vite 기반 정적 데모사이트에 적용됩니다. 루트 `AGENTS.md`와 `docs/AGENTS.md`의 제품 범위 지침을 함께 따릅니다.

## 역할

`demo/`는 로그인/백엔드 없이 Stology 핵심 화면 흐름을 체험하기 위한 정적 데모입니다. 실제 서비스 구현이 아니라 제품 기획과 화면 요구사항을 검증 가능한 프론트엔드 흐름으로 보여주는 산출물입니다.

## 근거 문서

데모 동작이나 화면 구조를 바꾸기 전에 아래 문서를 우선 확인합니다.

1. `../docs/기획안.md`
2. `../docs/변경사항/합의_0514.md`
3. `../docs/wire_frame_requirements/00-index.md`
4. 변경 대상 화면의 `../docs/wire_frame_requirements/0X-*.md`
5. `docs/traceability.md`

## 주요 명령

```bash
npm install       # 최초 의존성 설치
npm run dev       # 로컬 미리보기
npm run typecheck # TypeScript 검사
npm run lint      # ESLint 검사
npm run build     # 프로덕션 빌드
```

## 구현 경계

- 로그인은 구현하지 않고 로그인된 Home 화면에서 시작합니다.
- 백엔드 API, 브라우저 저장소, 실제 파일 업로드/파싱은 추가하지 않습니다.
- 네트워크 연동, 인증 플로우, 외부 문서 연동, 실제 계정/실명 데이터는 추가하지 않습니다.
- 데모 데이터는 `사용자1`, `스터디1`, `자료1`, `개념1`처럼 추상 명칭만 사용합니다.
- 인터랙션은 정적 상태 전환으로 구현하고, 실제 저장/업로드/분석처럼 오해될 수 있는 동작은 만들지 않습니다.

## UI/UX 기준

- 시각 스타일은 화이트/그레이 스켈레톤으로 유지하고 의미 색상은 쓰지 않습니다.
- 주요 화면/상호작용 컴포넌트에는 서비스 기능과 개발/디자인 요구사항을 담은 hover/focus 말풍선을 유지합니다.
- 3D 온톨로지 그래프는 `react-force-graph-3d` 기반으로 유지하되 그래프 직접 편집 기능은 추가하지 않습니다.
- 텍스트가 버튼, 카드, 탭, 패널 안에서 넘치거나 겹치지 않게 모바일/데스크톱 모두 고려합니다.
- 반복되는 화면 구조는 기존 컴포넌트와 스타일 토큰을 우선 재사용합니다.

## 코드 작업 원칙

- 새 의존성은 명시 요청이 있거나 기존 구현으로 해결하기 어렵다는 근거가 있을 때만 추가합니다.
- 데모 상태와 데이터 계약을 바꾸면 `src/types/`, `src/data/`, `src/state/`의 관계를 함께 확인합니다.
- 화면 흐름이나 요구사항 매핑이 바뀌면 `docs/traceability.md`를 함께 갱신합니다.
- `node_modules/`, `dist/`, `.tmp/`, `*.tsbuildinfo`, `.DS_Store`는 산출물로 취급하지 않습니다.
- 사용자가 수정 중인 파일을 되돌리지 말고, 현재 변경 위에 필요한 부분만 작게 반영합니다.

## 검증

데모 변경 후 변경 범위에 맞게 아래를 실행합니다.

```bash
npm run typecheck
npm run lint
npm run build
```

UI 흐름이나 그래프 동작을 바꾼 경우에는 로컬 미리보기 또는 브라우저 스모크 확인으로 주요 흐름을 확인합니다. 검증을 실행하지 못하면 완료 보고에 이유와 대체 확인을 남깁니다.
