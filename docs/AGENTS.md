# Stology 문서 작업 지침

이 파일은 `docs/` 아래 제품 기획, 화면설계서, 변경 이력, 와이어프레임 요구사항 문서에 적용됩니다. 루트 `AGENTS.md`의 공통 프로젝트 관리 지침을 함께 따릅니다.

## 역할

`docs/`는 Stology의 제품 범위와 화면 요구사항의 기준점입니다. 데모사이트, 영상, Figma/PM 산출물은 이 폴더의 기획과 화면 명세를 근거로 삼습니다.

## 우선 확인 순서

제품/화면 관련 작업은 아래 순서로 근거를 확인합니다.

1. `기획안.md`
2. `변경사항/합의_0514.md`
3. `wire_frame_requirements/00-index.md`
4. 필요한 화면별 `wire_frame_requirements/0X-*.md`
5. PM 산출물 맥락이 필요할 때 `화면설계서.md`

## 작성 위치

- 새 제품 문서는 `docs/` 아래에 둡니다.
- 화면 단위 요구사항은 `docs/wire_frame_requirements/`에 `NN-kebab-name.md` 형식으로 추가합니다.
- 변경 합의나 결정 기록은 날짜가 보이도록 `docs/변경사항/` 아래에 남깁니다.
- 기존 화면의 파생 모달/팝업/인라인 상태는 가능하면 해당 화면 문서 안에 함께 정리합니다.

## 편집 원칙

- MVP 범위에 없는 기능/화면/정책은 본문 요구사항으로 섞지 말고 “제안” 또는 “추후 확장”으로 분리합니다.
- 화면 목록, 번호, 흐름을 바꾸면 `wire_frame_requirements/00-index.md`도 함께 갱신합니다.
- 문서 사이의 파일명, 화면 번호, 권한 정책, 진입 경로가 서로 어긋나지 않게 확인합니다.
- 기존 표 형식과 섹션 구조를 유지해 PM 와이어프레임 제작자가 화면별 문서 하나만 보고 작업할 수 있게 합니다.
- 오래된 문서명을 참조해야 하는 경우, 현재 파일명과의 관계를 짧게 밝혀 추적성을 남깁니다.

## 용어

문서에서는 Stology 핵심 용어를 일관되게 사용합니다.

- `Stology`
- `Study + Ontology`
- `온톨로지`
- `자료`
- `지식 구조`
- `노드`
- `그래프`
- `Evidence`
- `검토`
- `합의`
- `주차별 리포트`

## 검증

문서 변경 후 최소한 다음을 확인합니다.

- 새로 추가/수정한 링크와 파일명이 실제 경로와 맞는지 확인
- 화면 번호와 진입 흐름이 `wire_frame_requirements/00-index.md`와 맞는지 확인
- MVP/보류/추후 확장 범위가 섞이지 않았는지 확인
- 변경 이유가 합의 사항이라면 `변경사항/`에 기록이 남아 있는지 확인
