# 02. 홈 화면

> 와이어프레임 제작용 디자인 및 기능 요구사항
> 파생 화면: 스터디 생성 모달, 초대 링크 생성 모달 (생성 직후 진입 케이스)

---

## 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 홈 화면 |
| 화면 유형 | 풀스크린 페이지 |
| URL | `/` 또는 `/home` |
| 권한 | 로그인 사용자 |
| 진입 경로 | 카카오 로그인 직후 / 사이드바 "홈" 클릭 / 초대 링크 모달 닫기 시 |

---

## 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────┐
│  [사이드바]              │    [우측 상단 프로필]              │
│                          │                                   │
│  - 홈                    │   ┌─ 진입점 섹션 ──────────────┐   │
│  - 스터디                 │   │ 스터디 (가로 1열, 스크롤)    │   │
│    └ 스터디 A             │   │ [카드A][카드B][카드C][+카드] │   │
│    └ 스터디 B             │   └────────────────────────────┘   │
│    └ 지난 활동             │                                   │
│      └ 종료 스터디 X       │   ┌─ 활동 영역 ─────────────────┐  │
│                          │   │ [개인 활동 로그][팀원 활동]   │   │
│                          │   └────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 영역별 비율 가이드
- 좌측 사이드바: 약 20% 너비, 항상 펼침
- 우측 상단 프로필: 우상단 고정
- 메인 영역: 진입점 섹션 + 활동 영역 (위/아래 분할)
- 활동 영역: 좌우 50:50 분할

---

## 화면 구성 요소

### 1. 좌측 사이드바

#### 구조
```
- 홈                          ← 현재 위치
- 스터디
  - [참여 중인 스터디 1]
  - [참여 중인 스터디 2]
  - 지난 활동
    - [종료된 스터디 1]
    - [종료된 스터디 2]
```

#### 규칙
- **항상 펼침 상태 유지** (펼침/접힘 토글 없음)
- 상위 메뉴("스터디")는 클릭 안 됨 (라벨 역할)
- 현재 페이지 시각 표시 (홈에 있을 때는 "홈" 강조)
- 종료된 스터디는 "지난 활동" 섹션에서 접근

### 2. 우측 상단 프로필
- 카카오 프로필 사진 (원형)
- 이름
- 닉네임 (선택)
- 스터디장 배지 등 별도 표시 없음

### 3. 진입점 섹션 — 스터디

#### 레이아웃
- 가로 1열, 가로 스크롤
- 카드 형태 나열
- 맨 우측에 **+ 카드** (항상 노출)

#### 일반 스터디 카드 구성
- 스터디명
- 현재 주차 (시작일 기준 자동 계산)
- NEW 배지 (최근 1일 이내 자료 업로드 있을 시)

#### + 카드 구성
- 시각적으로 구분되는 카드 (예: 점선 테두리, + 아이콘)
- 라벨: "스터디 생성" 또는 "+ 새 스터디"
- 진행 중 스터디 0개일 때도 항상 노출

#### 클릭 동작
| 클릭 위치 | 동작 |
|---|---|
| 일반 스터디 카드 | 해당 스터디 페이지로 이동 (`/study/{id}`) |
| + 카드 | 스터디 생성 모달 호출 |

#### 표시 범위
- 진행 중인 스터디만
- 종료된 스터디는 미표시 (사이드바 "지난 활동"에서 접근)

### 4. 활동 영역 (좌우 분할)

#### 좌측: 개인 활동 로그
- 본인의 활동 시간순 정렬
- 예시 항목:
  - "X 스터디에 자료 'OOO' 업로드"
  - "Y 스터디 검토 완료"
  - "Z 스터디에 질문 'OOO' 작성"

#### 우측: 팀원 활동 로그
- 참여 중인 스터디의 다른 팀원 활동 시간순 정렬
- 예시 항목:
  - "X 스터디: 김철수님이 자료 'OOO' 업로드"
  - "Y 스터디: 이영희님이 답글 작성"

---

## 기능 요구사항

### 진입점 카드 동작

**일반 스터디 카드 클릭**
- 해당 스터디 페이지 진입 (기본 탭은 지식 구조 탭)

**NEW 배지 표시 규칙**
- 최근 1일 이내 자료 업로드가 있을 시 노출
- 본인 업로드 포함 여부: MVP 미정의 (스터디 단위로 판단)

**가로 스크롤**
- 카드 수가 화면 너비를 초과하면 가로 스크롤
- + 카드는 항상 맨 우측 (스크롤 시 따라옴 또는 끝에 위치)

### 스터디 생성 진입

**+ 카드 클릭 시**
- 스터디 생성 모달 호출 (아래 파생 화면 참조)

### 활동 로그 동작

- 시간순 정렬 (최신순)
- 클릭 시 해당 활동의 출처 (스터디 페이지 등)로 이동
- 페이지네이션 또는 무한 스크롤: MVP 미정의

---

## 인터랙션 명세

| 인터랙션 | 동작 |
|---|---|
| 스터디 카드 클릭 | 해당 스터디 페이지 진입 |
| + 카드 클릭 | 스터디 생성 모달 오픈 |
| 사이드바 "홈" 클릭 | 현재 페이지 유지 (또는 새로고침) |
| 사이드바 스터디 이름 클릭 | 해당 스터디 페이지 이동 |
| 사이드바 "지난 활동" 종료 스터디 클릭 | 종료된 스터디 페이지 (조회 모드) |
| 활동 로그 항목 클릭 | 해당 출처로 이동 |

---

## 빈 상태

| 케이스 | 처리 |
|---|---|
| 참여 중인 스터디 0개 | "참여 중인 스터디가 없습니다" 메시지 + 항상 보이는 + 카드 |
| 개인 활동 0개 | (MVP 미정의) |
| 팀원 활동 0개 | (MVP 미정의) |

---

## 디자인 가이드

### 시각 우선순위
1. 진입점 섹션 (스터디 카드) — 가장 눈에 띄게
2. + 카드 — 시각적으로 구분되되 일반 카드와 같은 영역
3. 활동 영역 — 보조 정보

### 카드 디자인
- 일반 카드: 솔리드 배경, 그림자 등
- + 카드: 점선 테두리, 중앙에 + 아이콘 (시각적 구분)
- NEW 배지: 카드 우상단 모서리에 작은 배지

### 시각적 강조
- NEW 배지: 강조 색상 사용

# 파생 화면 1. 스터디 생성 모달

## 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 스터디 생성 모달 |
| 화면 유형 | 모달 (오버레이) |
| 진입 경로 | 홈 화면 진입점 카드 영역 맨 우측 + 카드 클릭 |
| 권한 | 로그인 사용자 누구나 |

## 레이아웃 구조

```
┌─────────────────────────────────────┐
│  스터디 생성             [X]         │
├─────────────────────────────────────┤
│                                     │
│  스터디 이름 *                       │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  └─────────────────────────────┘    │
│  (최대 20자)                         │
│                                     │
│  온톨로지 템플릿 *                    │
│  ┌─────────────────────────────┐    │
│  │ 선택...                   ▼ │    │
│  └─────────────────────────────┘    │
│                                     │
│  시작일 *                            │
│  ┌─────────────────────────────┐    │
│  │ YYYY-MM-DD                  │    │
│  └─────────────────────────────┘    │
│                                     │
│  설명                                │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  │                             │    │
│  └─────────────────────────────┘    │
│  (최대 200자, 선택)                  │
│                                     │
│              [생성하기]              │
└─────────────────────────────────────┘
```

## 입력 필드

| 필드 | 필수 | 검증 | UI |
|---|---|---|---|
| 스터디 이름 | 필수 | 최대 20자, 중복 허용 | 단일 라인 텍스트 입력 |
| 온톨로지 템플릿 | 필수 | 서비스 제공 템플릿 중 선택 | 드롭다운 (이름만 표시) |
| 시작일 | 필수 | YYYY-MM-DD, 과거 날짜 허용 | 텍스트 입력 |
| 설명 | 선택 | 최대 200자 | 여러 줄 텍스트 입력 |

## 액션

| 버튼 | 동작 | 활성화 조건 |
|---|---|---|
| 생성하기 | 스터디 생성 → 초대 링크 모달 전환 | 모든 필수 필드 입력 시 활성화 |
| X (닫기) | 모달 닫힘, 입력 내용 폐기 | 항상 활성 |

## 에러 처리
- 서버 오류, 네트워크 오류 등: **토스트 알림**
- 폼 검증 에러 (예: 글자 수 초과): 인라인 메시지 (필드 아래)

## 생성 완료 후 동작
- 스터디 생성됨
- **초대 링크 생성 모달로 자동 전환** (파생 화면 2 참조)
- 사이드바와 진입점 섹션에 새 스터디 카드 추가

# 파생 화면 2. 초대 링크 생성 모달 (생성 직후 진입)

## 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 초대 링크 생성 모달 (생성 직후 진입) |
| 화면 유형 | 모달 (오버레이) |
| 진입 경로 | 스터디 생성 모달의 "생성하기" 완료 후 자동 전환 |
| 권한 | 스터디장 (스터디 생성자) |

> 참고: 이 모달은 스터디 페이지의 ⚙ 설정에서도 진입 가능하나, 그 경우는 `08-study-settings.md` 문서에 정의됨

## 레이아웃 구조

```
┌─────────────────────────────────────┐
│  스터디 초대              [X]         │
├─────────────────────────────────────┤
│                                     │
│  ┌─────────────────────────────┐    │
│  │ https://stology.com/inv/... │    │
│  └─────────────────────────────┘    │
│                                     │
│              [복사하기]              │
│                                     │
│              [닫기]                  │
└─────────────────────────────────────┘
```

## 구성 (미니멀)

- 모달 제목: "스터디 초대"
- 초대 링크 URL (읽기 전용 텍스트 박스, 전체 선택 가능)
- 복사 버튼
- 닫기 버튼 (또는 X)

## 액션

| 버튼 | 동작 |
|---|---|
| 복사하기 | 클립보드에 URL 복사 + 토스트 알림 ("링크가 복사되었습니다") |
| 닫기 / X | 모달 닫힘 → 홈 화면 (배경에 새 스터디 카드 추가된 상태로 보임) |

## URL 형식
- 긴 토큰 (UUID 기반, 백엔드 구현 사항)
- 예: `https://stology.com/invite/{long-token}`

## MVP 제약
- 링크 재발급 불가 (1회 발급, 영구 유지)
- 만료 기간 없음
