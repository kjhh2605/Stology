### 영역 비율
- 상단: 자료 업로드 영역 (약 40%)
- 하단: 대기 중인 자료 영역 (약 60%, 스크롤 가능)

---

## 화면 구성 요소 — 상단: 자료 업로드 영역

### 1. 입력 방식 선택

- **파일 업로드** (드래그 앤 드롭 + 파일 선택 버튼)
- **텍스트 직접 입력** (텍스트 영역)
- 둘 중 하나 선택해서 사용

### 2. 파일 업로드 (드롭존)
- 드래그 앤 드롭 가능 영역
- "파일 선택" 버튼 (클릭으로 파일 다이얼로그)
- 허용 형식: **마크다운 파일만 (.md)**

### 3. 텍스트 직접 입력
- 여러 줄 텍스트 입력 영역
- 마크다운 형식 작성 가능

### 4. 입력 항목

| 필드 | 필수 | 설명 |
|---|---|---|
| 주차 선택 | 필수 | 드롭다운, 1주차 ~ 현재 주차 |
| 자료 제목 | 필수 | 단일 라인 텍스트 |
| 자료 설명 | 선택 | 여러 줄 텍스트 |

### 5. 등록 버튼
- 클릭 시 AI 추출 시작
- 업로드 후 토스트 알림 ("업로드 완료, 잠시 후 검토 가능")
- 업로드 후 그 자리에 머무름 (검토 화면 자동 이동 없음)

---

## 화면 구성 요소 — 하단: 대기 중인 자료 영역

### 자료 항목 표시 정보

각 자료 항목에 다음 정보 표시:
- 업로드일
- 자료 제목
- 제출자 (스터디 멤버 누구의 자료든 다 보임)
- 추출 상태 (배지)

### 추출 상태 종류 및 표시

| 상태 | 의미 | 시각 표현 | 클릭 동작 |
|---|---|---|---|
| 추출 중 | AI 추출 진행 중 | 회색 또는 노란색 배지, 로딩 아이콘 | 클릭 반응 없음 |
| 추출 실패 | AI 추출 실패 | 빨간색 배지 | 클릭 반응 없음 |
| 검토 필요 | AI 추출 완료, 검토 미완료 | 강조 배지 (파란색 등) | 검토 화면 진입 (신규) |
| 수정하기 | 본인이 검토 완료, 결과 미확정 | 다른 색상 배지 (보라색 등) | 검토 화면 재진입 (변경) |

### 정렬 순서
1. 검토 필요
2. 추출 실패
3. 추출 중

### 자동 처리
- 추출 실패 자료는 **1일 후 자동 삭제**

### 노출 범위
- 스터디 멤버 전원의 자료
- 본인 자료 + 다른 멤버 자료 모두 표시

## 기능 요구사항

### 업로드 흐름

```
[자료 업로드 영역에 입력]
   ↓ "등록" 클릭
[AI 추출 시작 (백그라운드)]
   ↓
[토스트 알림: "업로드 완료, 잠시 후 검토 가능"]
   ↓ (그 자리에 머무름)
[하단 대기 영역에 자료 추가, "추출 중" 상태]
   ↓ AI 처리 완료
[상태 → "검토 필요"로 변경]
```

### 검토 진입 흐름

```
[검토 필요 자료 클릭]
   ↓
[AI 추출 결과 검토 화면]
   ↓ "검토 마치기"
[자료 업로드 탭 복귀]
   ↓ (인원 충족 시) → 자료가 대기 영역에서 사라짐
   ↓ (인원 미충족 시) → "수정하기" 상태로 본인에게 노출
```

### 수정하기 흐름

```
["수정하기" 자료 클릭]
   ↓
[AI 추출 결과 검토 화면 (편집 모드)]
   ↓ 변경 후 "검토 마치기"
[자료 업로드 탭 복귀]
```

### 추출 실패 처리

- 자동 삭제 1일 카운트는 추출 실패 시점 기준
- 사용자 수동 삭제는 MVP 미정의

## 인터랙션 명세

### 업로드 영역
| 인터랙션 | 동작 |
|---|---|
| 파일 드래그 앤 드롭 | 드롭존에 파일 표시 |
| "파일 선택" 버튼 클릭 | 파일 다이얼로그 |
| 텍스트 직접 입력 모드 전환 | 텍스트 영역 노출 |
| 주차 드롭다운 클릭 | 1주차~현재 주차 목록 노출 |
| "등록" 버튼 클릭 | AI 추출 시작 + 토스트 |

### 대기 영역
| 인터랙션 | 동작 |
|---|---|
| "검토 필요" 자료 클릭 | AI 추출 결과 검토 화면 진입 |
| "수정하기" 자료 클릭 | AI 추출 결과 검토 화면 재진입 |
| "추출 중" 자료 클릭 | 반응 없음 |
| "추출 실패" 자료 클릭 | 반응 없음 |

---

## 빈 상태 / 에러 상태

| 케이스 | 처리 |
|---|---|
| 대기 자료 0개 | (MVP 미정의, 별도 안내 없음 추정) |
| 업로드 실패 | 토스트 에러 알림 |
| AI 추출 결과 0개 | 검토 화면에서 안내 ("스터디와 관련 있는 주제를 업로드해주세요") |

## 종료된 스터디 처리

| 영역 | 동작 |
|---|---|
| 자료 업로드 영역 | 비활성 또는 미노출 |
| 대기 중인 자료 영역 | 조회만 가능 (클릭 시 검토 화면 진입 불가) |
| "검토 필요" / "수정하기" 상태 | 클릭 반응 없음 |

## 디자인 가이드

### 상하 영역 구분
- 시각적 구분선 또는 영역 색상 차이
- 상단: 액션 중심 (입력 폼)
- 하단: 리스트 중심 (정보 조회)

### 상태 배지 색상
- 추출 중: 중립 또는 진행 (회색/노란색)
- 추출 실패: 위험 (빨간색)
- 검토 필요: 강조 (파란색 또는 메인 컬러)
- 수정하기: 보조 강조 (보라색 또는 다른 색)

### 입력 폼
- 필수 필드 표시 (*)
- 등록 버튼은 필수 필드 미입력 시 비활성 (검증 필요 — MVP 미명시이나 권장)

### 드롭존
- 점선 테두리
- 호버 / 드래그 오버 시 시각 변화
- 파일 형식 안내 ("마크다운 파일만 가능")
