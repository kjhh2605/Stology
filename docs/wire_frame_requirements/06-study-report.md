## 화면 구성 요소

### 1. 주차 토글 영역
- 토글 형식 (주차별 기록 탭과 동일 패턴)
- 1주차 ~ 현재 주차 노출
- 한 번에 하나 선택

### 2. 섹션 1: 진행 상황 요약

#### 시각 정보 1: 활성화 노드 수
- 큰 숫자 또는 카드 형태
- "이번 주 활성화 노드 수: N개"

#### 시각 정보 2: 도넛 차트
- **신규 활성화 노드 수 vs 보강 노드 수** 비율
- 두 색상으로 구분
- 차트 옆에 범례 (라벨 + 숫자)
- 예: 신규 활성화 5개 / 보강 7개

#### AI 텍스트
- 이번 주 학습 내용 요약 (AI 자동 생성)
- 한 문단 분량
- 주차별 업로드된 자료 기반

### 3. 섹션 2: 부족 개념

- AI 자연어 요약 (짧은 분량)
- 부족한 개념의 노드 리스트 (쉼표 구분)
- 둘 다 표시

### 4. 섹션 3: 팀 통계

#### 누적 막대그래프 (Stacked Bar)
- 팀원별 막대 1개
- **자료 업로드 수와 질문 작성 수를 색상으로 누적 표시**
- 색상으로 두 항목 구분
- 막대 옆에 팀원 이름과 수치

#### 팀원별 카드 (막대그래프 아래)
- 각 팀원당 카드 1개
- 카드 안에 AI 짧은 피드백 코멘트

## 기능 요구사항

### 리포트 생성 시점
- 주차 종료 시 자동 생성
- 사용자 수동 생성 액션 없음

### 리포트 단위
- 주차별 1개씩
- 누적 종합 리포트 없음

### 권한
- 스터디 멤버 누구나 조회 가능
- 권한별 표시 차이 없음

### 외부 내보내기
- 없음 (조회만)

### 점수 산식 (팀 통계 막대그래프)
- MVP는 **단순 카운트** (자료 업로드 수 + 질문 작성 수)
- 가중치 부여 등은 추후 결정

## 디자인 가이드

### 섹션 구분
- 카드 형태로 섹션 분리
- 섹션 제목 명확히 표시

### 시각 표현
- **숫자**: 큰 사이즈 강조 (활성화 노드 수)
- **도넛 차트**: 두 색상 명확히 구분, 범례 노출
- **막대그래프**: 누적 막대 색상으로 자료/질문 구분
- **AI 텍스트**: 인용문 스타일 또는 카드 내부 텍스트

### 색상 정책
- 도넛 차트: 신규 vs 보강 두 색상 (예: 메인 컬러 + 보조 컬러)
- 막대그래프: 자료 vs 질문 두 색상

### 팀원 카드
- 카드 그리드 또는 세로 나열
- 카드 안에 프로필 사진 + 이름 + AI 코멘트
