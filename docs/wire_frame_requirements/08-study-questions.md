## 화면 구성 요소

### 1. 헤더 영역

- 우측 상단: "질문 작성" 버튼 (모든 스터디 멤버 노출)
- 종료된 스터디에서는 "질문 작성" 버튼 미노출

### 2. 게시글 리스트

- 정렬: **최신순 고정** (필터/정렬 옵션 없음)
- 페이지네이션: 페이지 번호 방식 (1, 2, 3...)

### 3. 게시글 항목 (접힘 상태)

표시 정보:

| 항목 | 표시 위치 |
|---|---|
| 제목 | 좌측 |
| 작성자 (프로필 사진 + 이름) | 우측 |
| 작성일 | 우측 |
| 답글 수 | 우측 (💬 + 숫자) |
| 이미지 첨부 여부 | 우측 (📎 아이콘, 첨부 있을 때만) |

- 본인 작성 게시글에 별도 시각 표시 없음 (작성자명으로 구분)

### 4. 게시글 클릭 → 인라인 펼침

- 같은 화면에서 펼침/접힘
- 별도 상세 화면 이동 없음
- 다시 클릭 시 접힘

### 5. 페이지네이션

- 페이지 번호 클릭 시 해당 페이지로 이동
- 페이지당 게시글 수: MVP 미정의 (구현 시 결정)

### 6. 빈 상태

- 게시글 0개일 때:
  - 메시지: "아직 질문이 없습니다. 첫 질문을 작성해보세요!"
  - 일러스트 (선택)

# 파생 화면 1. 게시글 인라인 펼침 영역

## 개요

| 항목 | 내용 |
|---|---|
| 화면 유형 | 인라인 펼침 영역 |
| 진입 | 게시글 항목 클릭 |
| 닫기 | 같은 게시글 항목 재클릭 |

## 펼침 영역 구성 (위에서 아래)

### 1. 질문 본문 영역

```
[질문 본문 텍스트]
- 텍스트 (최대 1000자, 마크다운 미지원)
- 본문 안에 인라인 이미지 (개수 제한 없음)
```

- 본인 작성 질문일 경우: 우측에 수정/삭제 버튼 노출
- 다른 사람 질문일 경우: 수정/삭제 버튼 미노출

### 2. 답글 작성 영역 (항상 노출, 답글 영역보다 위)

```
[답글 작성 입력창]
┌──────────────────────────────────┐
│ 답글을 입력하세요...             │
└──────────────────────────────────┘
[📎 이미지 첨부]      [답글 작성]
```

- 텍스트 입력창: 본문 입력 (최대 1000자)
- 이미지 첨부 버튼: 최대 5개
- "답글 작성" 버튼: 본문 입력 시 활성화

### 3. 답글 리스트

```
─────────
답글 1 (가장 오래된 답글)
  [프로필 사진] 작성자 이름 · 작성일
  답글 본문 텍스트
  [첨부 이미지들]
  [본인 답글일 경우: 수정 / 삭제 버튼]
─────────
답글 2
...
```

- 정렬: 작성 시간 **오래된 순** (위에서 아래로 시간 흐름)
- 본인 답글에만 수정/삭제 버튼 노출

### 4. 빈 답글 상태

- 답글 0개 → **답글 영역 자체 숨김**
- 답글 작성 입력창만 노출

# 파생 화면 2. 질문 작성 모달

## 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 질문 작성 모달 |
| 화면 유형 | 모달 (오버레이) |
| 진입 경로 | 질문함 탭 헤더 우측 "질문 작성" 버튼 클릭 |
| 권한 | 스터디 멤버 누구나 |

## 레이아웃

```
┌─────────────────────────────────────┐
│  질문 작성                [X]         │
├─────────────────────────────────────┤
│                                     │
│  제목 *                              │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  └─────────────────────────────┘    │
│  (최대 50자)                         │
│                                     │
│  본문 *                              │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  │  (텍스트 + 인라인 이미지)    │    │
│  │                             │    │
│  │                             │    │
│  └─────────────────────────────┘    │
│  (최대 1000자)                       │
│                                     │
│              [질문하기]              │
└─────────────────────────────────────┘
```

## 입력 필드

| 필드 | 필수 | 검증 | UI |
|---|---|---|---|
| 제목 | 필수 | 최대 50자 | 단일 라인 텍스트 입력 |
| 본문 | 필수 | 최대 1000자 (텍스트만 카운트) | 여러 줄 텍스트 + 이미지 인라인 |

## 본문 입력 방식

- **단순 텍스트 + 이미지 인라인 붙여넣기 (Ctrl+V)**
- 일반 블로그 글 작성 느낌
- 마크다운 미지원 (`**bold**`, `#` 등 입력해도 그대로 표시됨)
- 텍스트 서식(볼드, 이탤릭, 헤딩 등) 없음
- 이미지 개수 제한 없음
- 이미지는 본문 안에 직접 보임 (별도 미리보기 영역 없음)
- 이미지 위치는 본문 안에 자유롭게 배치 가능

## 액션

| 버튼 | 동작 | 활성화 조건 |
|---|---|---|
| 질문하기 | 질문 등록 → 모달 닫힘 → 리스트 최상단에 새 질문 추가(접힘 상태) | 제목 + 본문 모두 입력 시 활성화 |
| X (닫기) | 모달 닫힘, **입력 내용 사라짐 (임시 저장 없음)** | 항상 활성 |

## 에러 처리

- 서버 오류, 네트워크 오류 등: **토스트 알림**

## 작성 완료 후 동작

- 모달 자동 닫힘
- 리스트 최상단에 새 질문 추가 (**접힘 상태**)
- 자동 펼침 없음

# 파생 화면 3. 질문 수정 모달 (편집 모드)

## 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 질문 작성 모달 (편집 모드) |
| 화면 유형 | 모달 (오버레이) |
| 진입 경로 | 본인 작성 질문의 인라인 펼침 영역 → "수정" 버튼 클릭 |
| 권한 | 본인 작성 질문에 한함 |

## 파생 화면 2와의 차이

질문 작성 모달과 동일한 UI, 단 다음 부분만 다름:

| 항목 | 신규 작성 모드 | 편집 모드 |
|---|---|---|
| 모달 제목 | "질문 작성" | **"질문 수정"** |
| CTA 라벨 | "질문하기" | **"수정하기"** |
| 입력 필드 초기값 | 빈 상태 | **기존 질문 내용으로 채워짐** |
| 작성 완료 후 동작 | 리스트 최상단에 새 질문 추가 | 기존 게시글 내용 갱신, 펼침 상태 유지 |

## 그 외 동작

- 입력 필드 검증, 본문 입력 방식, 에러 처리 등 신규 작성 모드와 동일

## 기능 요구사항

### F-1. 게시글 표시

- 최신순 정렬 (생성 시간 역순)
- 페이지네이션 (페이지 번호 방식)
- 페이지당 게시글 수는 MVP 미정의

### F-2. 게시글 클릭 인라인 펼침

- 같은 화면에서 펼침/접힘
- 동시에 여러 게시글 펼침 가능 (또는 단일만 펼침 — 구현 시 결정)
- 다시 클릭 시 접힘

### F-3. 질문 작성

- 헤더의 "질문 작성" 버튼 클릭 → 질문 작성 모달
- 작성 완료 후 리스트 최상단에 새 질문이 접힘 상태로 추가
- 자동 펼침 없음

### F-4. 답글 작성

- 게시글 펼침 영역의 답글 작성 입력창에서 작성
- "답글 작성" 버튼 클릭 → 답글이 답글 리스트 맨 아래에 추가 (오래된 순 정렬 유지)

### F-5. 수정/삭제 권한

- **본인 작성건만 수정/삭제 가능**
- 다른 사람 작성건에는 수정/삭제 버튼 미노출

### F-6. 수정 동작

- **질문 수정**: 별도 모달 (편집 모드)
- **답글 수정**: 인라인 편집

### F-7. 삭제 동작

- 질문/답글 모두 확인 모달 → 삭제

### F-8. 종료된 스터디 처리

- "질문 작성" 버튼 미노출
- 답글 작성 입력창 미노출
- 본인 작성건도 수정/삭제 불가 (버튼 미노출)
- 게시글/답글 조회만 가능

## 디자인 가이드

### 게시글 항목 시각 처리
- 접힘 상태: 간결한 행 형태
- 펼침 상태: 명확한 구분선으로 영역 분리
- 본인 작성건: 별도 시각 강조 없음

### 펼침 영역 시각 처리
- 질문 본문 영역 / 답글 작성 영역 / 답글 리스트를 명확히 구분 (구분선 또는 배경색)
- 답글 작성 영역은 항상 답글 리스트보다 위에 위치 (펼침 시 사용자가 바로 답글 작성 가능)

### 이미지 표시
- 인라인 본문 이미지: 본문 내 적절한 너비로 표시
- 답글 첨부 이미지: 답글 본문 아래 또는 본문 안에 표시

### 페이지네이션
- 페이지 번호 클릭 영역 명확화
- 현재 페이지 시각 강조
