# 09. AI 추출 결과 검토 화면

> 와이어프레임 제작용 디자인 및 기능 요구사항

---

## 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | AI 추출 결과 검토 화면 |
| 화면 유형 | 풀스크린 페이지 |
| URL | `/study/{id}/review/{materialId}` |
| 권한 | 스터디 멤버 (스터디장 + 스터디원, 본인 자료 본인 검토 가능) |
| 진입 경로 | 자료 업로드 탭 → 자료 항목 클릭 ("검토 필요" 또는 "수정하기" 상태) |

---

## 디자인 컨셉

**GitHub PR의 작업 단위 리뷰 패턴을 노드 단위로 변환**

- PR의 reviewer 영역 → 노드별 승인/반려 멤버 표시
- PR의 진행률 표시 → 상단 배너 (X/N 검토 완료)
- PR의 Submit Review 버튼 → "검토 마치기" 버튼

## 화면 구성 요소

### 1. 상단: 자료 메타 + 진행률 배너

#### 자료 메타 정보
- 자료 제목
- 업로더
- 업로드일
- 주차

#### 진행률 배너
- 형식: **"6/10 검토 완료"** 또는 유사 표현
- 본인 검토 기준 진행률 (모든 노드 후보 중 본인이 액션을 취한 비율)

### 2. 노드 후보 리스트

각 노드 후보가 하나의 카드 형태로 표시.

#### 노드 후보 카드 구성

**좌측 영역 (노드 정보)**
- 노드명
- AI 매칭 근거 (자료 원문에서 매칭된 부분 또는 AI의 추론)

**우측 영역 (Reviewer 영역, PR 패턴)**
- 현재 검토 상태 (X/N명 승인 또는 활성화 임박 등)
- 승인한 사람 (이름 또는 프로필 사진)
- 반려한 사람 (이름 또는 프로필 사진)

**액션 버튼 (각 노드별)**
- ☐ 체크박스 (일괄 처리용)
- "승인" 버튼
- "반려" 버튼

#### 본인 검토 시각 표시
- 본인이 이미 승인한 노드: 승인 버튼 강조 표시 (예: 채워진 상태)
- 본인이 이미 반려한 노드: 반려 버튼 강조 표시
- 본인이 미검토한 노드: 두 버튼 모두 기본 상태

### 3. 하단: 일괄 액션 + 검토 마치기

- "전체 승인" 버튼: 모든 노드 후보를 일괄 승인
- "선택 일괄 처리" 영역: 체크박스로 선택한 항목들에 대해 일괄 승인/반려
- "검토 마치기" 버튼: 검토 결과 제출

## 기능 요구사항

### F-1. 검토 액션

- 노드 후보별로 **승인 / 반려** 2가지 액션
- 수정 액션 없음
- 코멘트 작성 기능 없음
- 클릭 시 즉시 본인 검토 상태 변경 (별도 저장 없이 임시 저장)

### F-2. 본인 검토 변경

- "검토 마치기" 클릭 전까지 본인의 검토 결과 변경 자유
- "검토 마치기" 클릭 후에도 결과 확정(인원 충족) 전까지는 변경 가능
  - 자료 업로드 탭에서 해당 자료가 "수정하기" 상태로 노출됨
  - 클릭 시 본 화면 재진입하여 변경 가능
- 결과 확정 후에는 변경 불가

### F-3. 검토 마치기

- **모든 노드 후보에 액션을 취해야** "검토 마치기" 버튼 활성화
- 일부 노드만 검토 시 버튼 비활성
- 클릭 시:
  - 본인 검토 결과 제출
  - 자료 업로드 탭으로 복귀
  - 인원 미충족 시 자료 업로드 탭에서 "수정하기" 상태로 노출
  - 인원 충족 시 노드 활성화

### F-4. 일괄 액션

- **전체 승인**: 모든 노드 후보를 일괄 승인
  - 클릭 시 모든 노드의 본인 검토 상태가 "승인"으로 변경
- **선택 일괄 처리**:
  - 각 노드 카드의 체크박스로 선택
  - 선택된 항목에 대해 일괄 승인/반려 액션 (구체 UI는 디자인 단계에서 결정)

### F-5. 검토 통과 정책

- 스터디장이 설정한 인원 수(기본 2명) 이상이 승인 시 활성화
- 한 명이라도 반려하면 활성화되지 않음
- 본인이 업로드한 자료의 본인 검토 가능 (1명 카운트 인정)

### F-6. 빈 상태

| 케이스 | 처리 |
|---|---|
| AI 추출 결과 0개 | "스터디와 관련 있는 주제를 업로드해주세요" 안내 표시 |

### F-7. 동시성

- MVP 미고려
- 다른 멤버가 동시에 검토 중인 경우 마지막 저장만 반영

## 인터랙션 명세

| 인터랙션 | 동작 |
|---|---|
| 노드 후보의 "승인" 버튼 클릭 | 본인 검토 상태 "승인"으로 변경 (즉시 반영) |
| 노드 후보의 "반려" 버튼 클릭 | 본인 검토 상태 "반려"로 변경 (즉시 반영) |
| 본인이 이미 액션한 버튼 재클릭 | 해당 액션 취소 또는 다른 액션으로 전환 (구현 시 결정) |
| 체크박스 클릭 | 일괄 처리용 선택 토글 |
| "전체 승인" 클릭 | 모든 노드를 일괄 승인 |
| "선택 일괄 처리" | 선택된 항목에 대해 승인/반려 일괄 처리 |
| "검토 마치기" 클릭 | 검토 결과 제출 → 자료 업로드 탭으로 복귀 |
| 모든 노드 미검토 상태에서 "검토 마치기" 클릭 시 | 버튼 비활성 (클릭 자체 불가) |

## 디자인 가이드

### GitHub PR 디자인 참고 요소

| GitHub PR 요소 | 본 화면 매핑 |
|---|---|
| PR 헤더 (제목 등) | 상단 자료 메타 정보 |
| Files Changed 리스트 | 노드 후보 카드 리스트 |
| Reviewer 영역 (우측) | 각 노드 카드의 우측 (검토 상태 + 멤버) |
| 진행률 ("All checks passed") | 상단 진행률 배너 |
| Submit Review 버튼 | "검토 마치기" 버튼 |
| 일괄 액션 ("Approve all") | "전체 승인" / "선택 일괄 처리" |

### 시각 우선순위

1. 노드 후보 리스트 (메인 컨텐츠)
2. "검토 마치기" 버튼 (액션 마무리)
3. 진행률 배너 (현재 상태 인지)
4. 일괄 액션 (편의 기능)

### 노드 카드 시각 처리

- 본인이 이미 액션한 노드: 시각적으로 구분 (예: 좌측 색상 바, 배경색 변화 등)
- 미검토 노드: 기본 상태
- 인원 충족된 노드 (활성화 임박): 추가 시각 표시 (선택)

### 승인/반려 버튼

- 승인 버튼: 긍정 색상 (초록 또는 브랜드 강조 색상)
- 반려 버튼: 부정 색상 (빨강 또는 경고 색상)
- 본인 선택 상태는 명확히 구분

### 진행률 배너

- 화면 상단에 고정
- 본인 검토 진행률 (X/N 검토 완료)
- 시각적 진행률 바 추가 가능

---

## 본인 자료 본인 검토에 대한 안내

- 본인이 업로드한 자료의 검토에 본인도 참여 가능
- 본인 검토 시 일반 검토와 동일하게 1명 카운트
- 별도 UI 차이 없음
