## 1. 서비스 소개

### 서비스명

**Stology : 당신의 활동이 소중한 자산이 되도록**

### 한 줄 소개

Stology는 스터디에서 쌓이는 다양한 자료들을 활용해 체계적인 스터디 운영 플로우를 제공합니다

### 서비스 정의

스터디가 진행되는 동안 팀원들은 정리 노트, 발표자료 요약, 질문, 오답노트, 논문 요약 등 다양한 학습자료를 남깁니다.

하지만 이러한 학습자료는 대부분 개인 문서나 채팅, 저장소에 흩어져 있어 다음과 같은 문제가 발생합니다.

- 스터디장이 매주 학습자료를 읽고 핵심 개념과 질문을 직접 종합해야 합니다.
- 스터디원이 처음 배우는 개념이 다른 개념과 어떻게 연결되는지 파악하기 어렵습니다.
- AI가 정리한 개념과 관계를 검증 없이 사용하면 잘못된 학습 흐름이 만들어질 수 있습니다.
- 공통 학습자료를 체계적으로 관리하려면 팀원 모두가 고급 지식관리 도구를 이해해야 합니다.
- 질문과 보강 필요 개념이 다음 주차 자료로 자연스럽게 이어지지 않습니다.

Stology는 이 문제를 해결하기 위해 다음과 같은 스터디 운영 플로우를 제공합니다.

1. 스터디 생성 및 스터디원 모집 (초대 링크)
2. 학습자료 입력
3. AI 개념 후보 추출
4. GitHub PR 방식을 참고한 팀 검토 (승인/반려)
5. 공통 온톨로지 기반 지식 구조 확정
6. 질문함 관리
7. 커버리지 리포트 생성
8. 다음 스터디 자료 반영

### 인증 및 온톨로지 제공 방식

- 인증: 카카오 소셜 로그인 단일 방식을 사용합니다. 모든 페이지는 로그인 필수입니다.
- 공통 온톨로지: 서비스가 카테고리별 온톨로지 템플릿을 제공합니다. 스터디장이 스터디 생성 시 템플릿을 선택하며, MVP에서는 노드와 관계를 사용자가 수정할 수 없습니다. 추후 확장에서 스터디장의 온톨로지 수정과 사용자 온톨로지 생성을 지원합니다.

## 2. 서비스 목표

### 핵심 목표

Stology는 학습자료를 분석하여 공통 온톨로지 기반의 지식 구조로 전환하고, 학습 중 발생하는 질문을 별도의 질문함에서 관리하여 반복되는 혼란과 정보 유실을 줄입니다.

또한 이번 주차의 활성화 노드 현황, 부족 개념, 팀원 활동을 분석하여 커버리지 리포트를 생성합니다.

### 해결하고자 하는 문제

- 흩어진 학습자료 → AI 기반 핵심 개념 후보 추출
- 질문 휘발·누락 → 별도 질문함에서 관리
- 스터디 운영이 스터디장 개인 역량 의존 → 체계적인 워크플로를 서비스차원에서 일관되게 제공
- 처음 배우는 개념의 단절 → 선행·연결·후속 개념 맥락 제공
- 고급 지식관리 툴 진입장벽 → 학습자료 중심의 쉬운 참여 흐름

### MVP 목표

MVP에서는 다음 세 가지가 실제로 가능한지 검증합니다.

1. 스터디에서 쌓이는 다양한 학습자료를 하나의 운영 흐름으로 모을 수 있는가?
2. AI가 학습자료에서 핵심 개념 후보를 추출하고, 팀 검토를 통해 신뢰 가능한 지식 구조로 확정할 수 있는가?
3. 주차별 활동 기반 커버리지 리포트가 스터디 운영에 도움이 되는가?

> MVP의 커버리지 리포트는 다음 5개 항목으로 구성합니다.
> - 이번 주 활성화 노드 수
> - 이번 주 신규 활성화 노드 수 / 보강 노드 수 (도넛 차트)
> - 이번 주 학습 내용 (AI 요약)
> - 부족 개념
> - 팀원 활동 (자료 업로드 수 + 질문 작성 수 누적 막대그래프 + AI 피드백)
>
> 질문과 다음 자료 반영 항목은 별도 질문함 및 다음 자료 운영 흐름에서 활용하며, 커버리지 리포트의 직접 표시 항목에는 포함하지 않습니다.

## 3. 제품 구조

Stology는 스터디에서 쌓이는 다양한 자료들을 활용해 체계적인 스터디 운영 플로우를 제공합니다.

### 3-1. 학습자료 중심 운영 플로우 제공

개인 정리, 발표자료, 읽기 자료, 실습 기록 등 흩어진 학습자료에서 AI가 핵심 개념 후보를 추출하고, 팀 검토를 거쳐 공통 온톨로지 기반 지식 구조로 확정합니다.

MVP에서 지원하는 스터디자료 입력 방식은 다음과 같습니다.

- 마크다운 파일 업로드
- 텍스트 직접 입력

그 외 문서 형식, 외부 링크, Notion, Google Docs, Discord, Slack 등은 추후 확장 대상으로 둡니다.

### 3-2. 스터디장 운영 부담 감소

스터디장이 별도의 도구를 사용하지 않고 서비스를 통해 스터디원의 학습상황, 질문등을 체계적으로 관리할 수 있습니다.

스터디 생성자는 자동으로 해당 스터디의 스터디장이 되며, 1개 스터디는 1명의 스터디장을 가집니다. 스터디장은 검토 인원 수 조정, 스터디 종료, 초대 링크 조회/공유 권한을 가집니다.

### 3-3. 스터디원 개념 맥락 파악과 질문 참여 지원

처음 배우는 개념의 선행 개념, 연결 개념, 다음 학습 방향을 보여주고, 모르는 점은 질문함에 남길 수 있게 합니다.

### 3-4. AI 결과의 팀 검토 기반 신뢰성 확보

AI가 만든 개념 후보를 바로 확정하지 않고, GitHub PR 방식을 참고한 팀 검토를 거쳐 승인된 내용만 공통 온톨로지 기반 지식 구조에 반영합니다.

이때 AI는 최종 판단자가 아니라 개념 후보 제안자입니다. 최종 확정은 팀 검토를 통해 이루어집니다.

팀 검토는 노드 단위로 이루어지며, 액션은 **승인 / 반려** 2가지만 제공합니다. 스터디장이 설정한 인원 수(기본 2명) 이상이 승인하면 노드가 활성화됩니다. 한 명이라도 반려하면 활성화되지 않습니다.

### 3-5. 질문함 기반 학습 신호 관리

질문은 개념 노드에 직접 연결하지 않고, 별도 질문함을 통해 관리합니다.

MVP에서는 질문함을 스터디별 게시글-답글 형태로 제공합니다.

- 스터디 멤버가 질문 게시글을 작성합니다.
- 스터디 멤버가 답글 형태로 답변합니다. (스터디장/스터디원 구분 없음)
- 질문과 답변은 스터디별 질문함에 누적됩니다.
- 질문 입력 항목: 제목, 본문(텍스트 및 이미지 첨부 가능, 마크다운 미지원)

추후 확장에서는 답변에 포함된 단어와 온톨로지 노드를 매칭하고, 해당 단어를 하이퍼링크로 연결해 관련 노드의 자료 목록을 조회할 수 있게 합니다.

## 4. 서비스 배경

Stology는 스터디장과 스터디원을 위한 AI 기반 스터디 운영 플랫폼입니다.

### 4-1. 스터디장 페인포인트와 해결안

#### 문제 1. 자료와 질문이 노션, 카톡 등 여러 곳에 흩어져 있어 관리가 어렵다

- 현재 문제: 학습자료와 질문이 노션, 카카오톡, 개인 문서, 저장소 등 여러 곳에 분산됩니다.
- Stology 해결안: 학습자료와 질문을 하나의 스터디 공간에서 통합 관리합니다.

#### 문제 2. 스터디원이 어떤 개념에 대한 학습이 부족한지 한눈에 파악하기 어렵다

- 현재 문제: 학습자료와 활동 기록이 흩어져 있어 부족 개념을 운영자가 직접 판단해야 합니다.
- Stology 해결안: 학습자료를 공통 온톨로지 기반 지식 구조로 변환하고, 이번 주 학습 범위·부족 개념·팀원 활동 중심의 커버리지 리포트를 제공합니다.

#### 문제 3. AI가 만든 개념 관계를 검증 없이 쓰면 잘못된 지식 구조가 누적될 수 있다

- 현재 문제: AI가 생성한 개념과 관계가 항상 정확하다고 보기 어렵습니다.
- Stology 해결안: AI 분석 결과를 GitHub PR 방식을 참고해 스터디 단위에서 승인 또는 반려할 수 있게 합니다.

#### 문제 4. 다음 주차에 어떤 내용을 보강해야 할지 판단이 어렵다

- 현재 문제: 학습자료와 질문이 다음 자료 제작으로 자연스럽게 이어지지 않습니다.
- Stology 해결안: 학습자료, 부족 개념, 팀원 활동, 질문함 내용을 참고해 다음 자료에 반영할 항목을 판단할 수 있게 합니다.

#### 문제 5. 운영 방식과 노하우가 특정 스터디장 개인에게 의존한다

- 현재 문제: 좋은 운영 방식이 개인의 경험과 감각에 머무릅니다.
- Stology 해결안: 스터디 운영 플로우를 제품 구조로 표준화하여 팀 자산으로 축적합니다.

### 4-2. 스터디원 페인포인트와 해결안

#### 문제 1. 개념들이 서로 어떻게 연결되는지 파악하기 어렵다

- 현재 문제: 자료는 많지만 개념 간 관계나 학습 순서를 알기 어렵습니다.
- Stology 해결안: 공통 온톨로지 기반 지식 구조를 통해 관련 개념 추천과 지식 흐름 기반 탐색을 제공합니다.

#### 문제 2. 비슷한 질문이 반복되지만 이전 질문을 찾기 어렵다

- 현재 문제: 채팅이나 문서에 남긴 질문이 쉽게 묻힙니다.
- Stology 해결안: 게시글-답글 형태의 질문함을 통해 질문과 답변 기록을 관리합니다.

#### 문제 3. 고급 지식관리 도구는 진입장벽이 높아 참여 부담이 크다

- 현재 문제: 모든 팀원이 복잡한 지식관리 도구를 익히기는 어렵습니다.
- Stology 해결안: 도구 학습 없이 학습자료 등록, 질문 작성, 답글 확인 중심으로 간단히 참여할 수 있게 합니다.

#### 문제 4. 자신이 부족한 개념이나 학습 방향을 파악하기 어렵다

- 현재 문제: 무엇을 더 공부해야 하는지 개인이 스스로 판단해야 합니다.
- Stology 해결안: 커버리지 기반 학습 현황 및 보강 필요 영역을 확인할 수 있게 합니다.

### 핵심 관점

학습자료는 공통 온톨로지 기반 지식 구조로 정리하고, 질문은 별도 질문함에서 관리해야 스터디장의 운영 판단과 스터디원의 학습 참여가 다음 자료로 이어집니다.

## 5. Stology의 차별점

### 5-1. 일반 문서 / 워크스페이스

- 대표 서비스: Notion
- 강점: 문서 작성과 공유가 쉽고, 자료를 체계적으로 축적할 수 있습니다.
- 한계: 학습자료 기반 개념 분석, 질문 반영 운영 루프, 팀 기준 지식 확정 구조가 부족합니다.
- Stology 차별점: 문서 저장이 아니라 스터디 운영에 집중합니다. 학습자료, 질문, 커버리지 데이터를 연결해 다음 학습 운영까지 지원합니다.

### 5-2. 개인 지식관리 도구

- 대표 서비스: Obsidian, Tolaria
- 강점: 링크·백링크 기반 지식 연결과 개인 학습 구조화에 강합니다.
- 한계: 팀 전체가 사용법을 익혀야 하며, 스터디 운영 흐름과 질문 관리가 기본 제공되지 않습니다.
- Stology 차별점: 그래프 편집 지식 없이도 사용할 수 있습니다. 질문함 중심으로 운영 흐름을 단순화하고 팀 검토 기반 온톨로지 지식 구조를 제공합니다.

### 5-3. AI 문서 보조 도구

- 대표 서비스: NotebookLM, ChatGPT, Claude
- 강점: 빠른 요약·질의응답·설명 생성이 가능합니다.
- 한계: 팀 운영 흐름, 팀 검토 기반 지식 확정, 누적 학습 커버리지 관리가 약합니다.
- Stology 차별점: AI 응답 생성보다 팀 검토 후 운영 자료로 확정하는 흐름에 초점을 둡니다.

### 5-4. LMS / Classroom

- 대표 서비스: Google Classroom, Canvas
- 강점: 과제·제출·성적 관리에 강합니다.
- 한계: 자율 스터디 운영에는 무겁고, 개념 구조화와 다음 자료 추천 기능이 부족합니다.
- Stology 차별점: 성적 관리보다 다음 스터디 준비와 운영에 최적화되어 있습니다.

### 5-5. 커뮤니케이션 도구

- 대표 서비스: Discord, Slack, KakaoTalk
- 강점: 질문과 소통을 빠르게 남기기 쉽습니다.
- 한계: 질문이 묻히고 학습 맥락 구조화가 어렵습니다.
- Stology 차별점: 게시글-답글 형태의 질문함으로 질문 기록을 별도 관리하고, 추후 답변의 단어-노드 매칭을 통해 관련 자료 목록으로 연결할 수 있습니다.

### 5-6. Q&A / 게시판 도구

- 대표 서비스: Stack Overflow, GitHub Discussions
- 강점: 질문·답변 구조화와 검색에 강합니다.
- 한계: 질문이 학습자료, 온톨로지 지식 구조, 스터디 운영 흐름과 연결되지 않습니다.
- Stology 차별점: 질문을 별도 질문함에 기록하고, 추후 답변 내 단어와 노드를 하이퍼링크로 연결해 관련 자료 목록 조회까지 확장할 수 있습니다.

### 차별화 요약

Stology는 특정 개발 스터디나 특정 도구에 종속되지 않습니다. 학습자료와 질문이 반복적으로 발생하는 모든 스터디에 적용할 수 있습니다.

## 6. MoSCoW

### 첫 번째 목표

다양한 학습자료를 AI 후보 추출과 팀 검토를 통해 공통 온톨로지 기반 지식 구조로 확정하고, 질문함에 모인 질문은 게시글-답글 형태로 관리합니다.

초기 MVP 구현 범위는 아래 MoSCoW에 별도로 정리합니다.

### Must have: 반드시 필요함

- 카카오 소셜 로그인
- 스터디 생성 및 주차 관리
  - 스터디 생성자는 자동으로 스터디장
  - 스터디장은 스터디 종료 가능
- 초대 링크를 통한 스터디원 모집
  - 스터디장만 초대 링크 발급/공유 가능
- **스터디장/스터디원 권한 구분**
  - 스터디장 권한: 검토 인원 수 조정, 스터디 종료, 초대 링크 조회/공유
  - 스터디원 권한: 자료 업로드, 검토, 질문/답글 작성
- 학습자료 등록
  - 마크다운 파일 업로드
  - 텍스트 직접 입력
  - 그 외 입력 방식은 추후 확장
- AI 기반 핵심 개념 후보 추출
- GitHub PR 방식을 참고한 팀 검토
  - AI가 추출한 개념 후보 확인
  - **승인 또는 반려** (2가지 액션, 코멘트 기능 없음)
  - 스터디장 설정 인원 수(기본 2명) 이상 승인 시 노드 활성화
  - 한 명이라도 반려하면 활성화 안 됨
- 승인된 개념의 공통 온톨로지 기반 지식 구조 반영 (회색 반투명 → 활성화)
- 질문함 (스터디별)
  - 게시글 형태의 질문 작성 (제목, 본문, 이미지 첨부)
  - 답글 형태의 답변 작성
  - 질문·답변 목록 확인 (최신순)
  - 본인 작성건 수정/삭제
- 커버리지 리포트 (5개 항목)
  - 이번 주 활성화 노드 수
  - 이번 주 신규 활성화 노드 수 / 보강 노드 수 (도넛 차트)
  - 이번 주 학습 내용 (AI 요약)
  - 부족 개념
  - 팀원 활동 (자료 업로드 수 + 질문 작성 수 누적 막대그래프 + AI 피드백)

### Should have: 가급적 필요함

- 스터디원용 개념 맥락 설명
- 원본 학습자료와 구조화 View 분리

### Could have: 있으면 좋음

- 스터디장의 멤버 강제 제거
- 스터디장의 검토 강제 마감
- 공개/비공개 스터디 구분
- 스터디장 권한 이양 또는 공동 스터디장
- 개인별 이해도/질문 흐름 요약
- 개인별 부족 개념 표시
- 복습용 퀴즈/질문 초안 생성
- 다음 기수용 학습자료 패키지 생성

- 답변의 단어-노드 매칭
- 하이퍼링크를 통한 해당 단어 또는 노드의 자료 목록 조회
- Notion / Google Docs 링크 기반 학습자료 등록

- 스터디장의 온톨로지 수정
- 사용자 온톨로지 생성

### Won't have: 이번 프로젝트에서는 고려하지 않음

- 완전 자동 온톨로지 생성
- AI가 개념 구조를 최종 확정
- AI가 질문 답변을 최종 확정
- 질문을 자동으로 지식 노드로 전환

- 외부 AI의 지식 구조 수정 권한

  - 흩어진 학습자료를 핵심 개념 구조와 다음 자료 초안으로 전환할 수 있기 때문입니다.
  - GitHub PR 방식을 참고한 팀 검토로 AI 개념 후보를 신뢰 가능한 지식 구조로 확정할 수 있기 때문입니다.
- 서비스 사용 시나리오:
  - 스터디 생성 및 온톨로지 템플릿 선택

  - AI 개념 후보 확인
  - 팀 검토 (승인/반려)
  - 질문함 확인
  - 커버리지 리포트 확인
  - 다음 주차 자료 준비

- 구분: 스터디원
- 나이/성별: 20세 / 여

## 8. 최종 서비스 흐름

1. 사용자가 카카오 소셜 로그인으로 진입합니다.
2. 스터디장이 홈 화면의 + 카드를 통해 스터디를 생성합니다.
   - 스터디 이름, 온톨로지 템플릿(드롭다운), 시작일, 설명 입력
   - 생성 직후 초대 링크 모달 노출
3. 스터디원이 초대 링크를 통해 스터디에 참여합니다.
4. 팀원이 해당 주차의 학습자료를 등록합니다.
   - MVP에서는 마크다운 파일과 텍스트 직접 입력을 지원합니다.
5. AI가 학습자료에서 핵심 개념 후보를 추출합니다.
6. 팀은 GitHub PR 방식을 참고해 AI 후보를 노드 단위로 검토합니다.
   - 후보 확인
   - 승인 또는 반려
   - 스터디장 설정 인원 수(기본 2명) 이상 승인 시 활성화
7. 승인된 개념은 공통 온톨로지 기반 지식 구조에 반영됩니다 (회색 반투명 → 활성화).
8. 스터디 멤버가 학습 중 모르는 점을 질문함에 게시글 형태로 남깁니다.
9. 스터디 멤버가 답글 형태로 답변합니다. (스터디장/스터디원 구분 없음)
10. Stology는 학습자료와 활동 데이터를 바탕으로 커버리지 리포트를 생성합니다.
    - 이번 주 활성화 노드 수
    - 이번 주 신규 활성화 노드 수 / 이번 주 보강 노드 수 (도넛 차트)
    - 이번 주 학습 내용 (AI가 작성한 주차별 업로드 자료기반 요약)
    - 부족 개념
    - 팀원 활동 (개인별 자료 업로드 수 + 질문 작성 수 누적 막대그래프 + 개인별 AI 피드백)
11. 스터디장은 커버리지 리포트와 질문함을 참고해 다음 스터디 자료를 준비합니다.
12. 스터디 종료 시 스터디장이 종료 처리, 종료된 스터디는 조회만 가능합니다.
13. 추후 확장에서는 답변 내 단어와 온톨로지 노드를 매칭하고, 하이퍼링크를 통해 해당 노드의 자료 목록으로 연결합니다.

## 10. 현재 기준 최종 문장

Stology는 스터디에서 매주 쌓이는 다양한 학습자료를 AI가 핵심 개념 후보로 정리하고, GitHub PR 방식을 참고한 팀 검토(승인/반려)를 거쳐 서비스 제공 온톨로지 기반의 지식 구조와 다음 스터디 자료로 확정하는 범용 스터디 운영 도구입니다.

학습 중 발생하는 질문은 개념 노드에 바로 연결하지 않고 별도의 스터디별 질문함에서 게시글-답글 형태로 관리합니다. 추후에는 답변에 포함된 단어와 온톨로지 노드를 매칭하고, 하이퍼링크를 통해 해당 노드의 자료 목록을 조회할 수 있도록 확장합니다.

Stology의 핵심은 단순한 문서 저장이나 AI 요약이 아니라, 학습자료와 질문을 팀 검토 기반의 운영 흐름으로 전환해 스터디의 반복 운영을 더 쉽게 만드는 것입니다.
