## 3. 변경 사항 상세

### 3.1 홈 화면 (#2)

**유지되는 부분:**
- D 하이브리드 레이아웃 (상단 진입점 + 하단 활동 로그)
- 우측 상단 프로필
- 좌측 사이드바
- 카드 정보 (이름, 현재 주차, NEW 배지)
- 하단 좌우 분할 활동 로그

**변경되는 부분:**
- **진입점 카드 영역에 + 카드 추가** (스터디 생성 진입점, 맨 우측)
- **프로젝트 섹션 삭제** (프로젝트 개념 사라짐)
- **사이드바에서 프로젝트 메뉴 삭제**
- **사이드바에서 운영진 메뉴 평면 추가 부분 삭제**

**새로운 사이드바 구조:**
```
- 홈
- 스터디
  - [참여 중인 스터디 1]
  - [참여 중인 스터디 2]
  - 지난 활동
    - [종료된 스터디들]
```

### 3.2 스터디 페이지 컨테이너 (#3)

**유지되는 부분:**
- 진입 경로 (홈 카드 / 사이드바)
- 페이지 헤더 (스터디명, 진행 주차, 멤버, 시작일, ⚙ 설정)
- ⚙ 설정은 스터디장만 노출

**변경되는 부분:**
- **상단 네비게이션 4탭 → 5탭** (질문함 추가)
- **⚙ 설정 메뉴 확장**: 초대 링크 / 검토 인원 수 조정 / 스터디 종료 3개 항목

### 3.3 자료 업로드 탭 (#3-2)

**유지되는 부분:**
- 상단 자료 업로드 영역 + 하단 대기 자료 영역
- 추출 상태 (추출 중 / 추출 실패 / 검토 필요 / 수정하기)
- 정렬 순서
- 자동 삭제 (1일 후)

**변경되는 부분:**
- **입력 방식이 마크다운 파일 + 텍스트 직접 입력** (이전: 마크다운만)
- 텍스트 직접 입력은 기획안 명시 사항 반영

### 3.4 주차별 리포트 탭 (#3-4)

**유지되는 부분:**
- 주차 단위 자동 생성
- 토글 형식 주차 선택
- 미완료 주차 안내

**변경되는 부분:**
- **리포트 구성 변경** (3 섹션 → 3 섹션, 내용 일부 변경)
  - 섹션 1 (진행 상황 요약):
    - 이번 주 활성화 노드 수
    - 신규 활성화 / 보강 비율 도넛 차트
    - AI 텍스트 (이번 주 학습 내용 요약, 한 문단)
  - 섹션 2 (부족 개념): AI 자연어 요약 + 노드 리스트
  - 섹션 3 (팀 통계):
    - **누적 막대그래프** (자료 업로드 수 + 질문 작성 수, 색상 구분)
    - 팀원별 카드 + AI 짧은 피드백 코멘트

### 3.5 AI 추출 결과 검토 화면 (#9)

**유지되는 부분:**
- GitHub PR 디자인 컨셉
- 노드 단위 검토
- 일괄 액션 (전체 승인, 선택 일괄 처리)
- 진행률 배너
- "검토 마치기" 버튼
- 본인 검토 변경 (수정하기 상태)

**변경되는 부분:**
- **용어 변경**: "거절" → **"반려"**
- 검토 정책 명시화 (스터디장 설정 인원 수, 기본 2명)

### 3.6 질문함 탭 (#3-5)

**구성:**
- 메인 리스트 영역 + 인라인 펼침 영역으로 통합
- 헤더 우측에 "질문 작성" 버튼
- 페이지네이션 (페이지 번호)

**게시글 항목:**
- 제목, 작성자, 작성일, 답글 수 (💬 N), 이미지 첨부 여부

**인라인 펼침 영역:**
- 질문 본문
- 첨부 이미지
- 답글 작성 입력창 (항상 노출)
- 답글 리스트 (오래된 순)

**수정/삭제:**
- 본인 작성건만 노출
- 질문 수정: 별도 모달 (질문 작성 모달 편집 모드)
- 답글 수정: 인라인 편집
- 삭제: 확인 모달 후 삭제

## 4. 신규 화면 정의

### 4.1 스터디 생성 모달 (#4)

**진입**: 홈 화면 진입점 카드 영역 맨 우측 + 카드 클릭
**입력**:
- 스터디 이름 (필수, 최대 20자, 중복 허용)
- 온톨로지 템플릿 (필수, 드롭다운, 이름만)
- 시작일 (필수, YYYY-MM-DD 텍스트 입력, 과거 허용)
- 설명 (선택, 최대 200자)

**액션**: "생성하기" / X 버튼 (필수 미입력 시 비활성)
**에러**: 토스트 알림
**생성 직후**: 초대 링크 생성 모달로 자동 전환

### 4.2 초대 링크 생성 모달 (#5)

**진입**:
- 스터디 생성 직후 자동
- 스터디 페이지 ⚙ 설정 → 초대 링크 클릭

**구성 (미니멀)**:
- 모달 제목 ("스터디 초대")
- 초대 링크 URL (읽기 전용)
- 복사 버튼
- 닫기 버튼

**복사 동작**: 클립보드 복사 + 토스트 알림
**닫기 동작**:
- 생성 직후 진입 → 홈
- ⚙ 진입 → 스터디 페이지로 복귀

**재발급**: MVP에서 없음
**URL**: 긴 토큰 (UUID 등, 백엔드 사항)

### 4.3 스터디 페이지 헤더 ⚙ 설정 (#6)

**진입**: 스터디 페이지 헤더 ⚙ 아이콘 클릭 (스터디장만 노출)
**UI**: 드롭다운 메뉴
**항목 (자주 쓰는 순)**:
1. 초대 링크 → 초대 링크 생성 모달 호출
2. 검토 인원 수 조정 → 인라인 증감 버튼 (+/-), 즉시 적용 + 토스트
   - 변경은 새로 시작되는 검토에만 적용
3. 스터디 종료 → 스터디 종료 확인 모달 호출

### 4.4 스터디 종료 확인 모달 (#7)

**진입**: ⚙ 설정 → 스터디 종료 클릭
**구성**:
- 안내 메시지: "스터디를 종료하면 더 이상 자료 업로드, 검토, 질문 작성이 불가능합니다"
- Yes / No 버튼

**확인 시**: 스터디 종료 안내 화면으로 이동

### 4.5 스터디 종료 안내 화면 (#8)

**진입**: 스터디 종료 확인 직후 1회
**구성**:
- 종료 메시지
- 종료 요약 (총 활성 노드 수, 업로드된 자료 수, 작성된 질문 수)
- "홈 이동" 버튼

**재진입**: 불가 (이후 종료된 스터디 클릭 시 곧장 조회 모드 진입)

### 4.6 질문 작성 모달 (#11)

**진입**:
- 질문함 헤더 우측 "질문 작성" 버튼 (신규 작성)
- 본인 질문 수정 버튼 (편집 모드)

**입력**:
- 제목 (필수, 최대 50자)
- 본문 (필수, 최대 1000자, 텍스트만 카운트)
  - 단순 텍스트 + 이미지 인라인 붙여넣기 (Ctrl+V)
  - 마크다운 미지원
  - 텍스트 서식 없음
- 이미지 개수 제한 없음

**액션**: "질문하기" (편집 시 "수정하기") / X 버튼 (필수 미입력 시 비활성)
**작성 완료 후**: 모달 닫힘 + 새 질문이 리스트 최상단에 추가 (접힘 상태)
**모달 닫기**: 입력 내용 사라짐 (임시 저장 없음)
