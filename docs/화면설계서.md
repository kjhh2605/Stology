# Stology 화면설계서

> 산출물 목적: Figma `stology_wf`의 Mid-Fi 와이어프레임을 기준으로, 사용자가 서비스를 이용하는 흐름과 화면별 인터랙션을 PM/디자이너/개발자가 같은 기준으로 이해할 수 있게 정리한다.

---

## 0. 작성 기준

### 0-1. 화면설계서 구성

각 화면은 다음 3요소를 함께 관리한다.

| 구성 | 설명 |
|---|---|
| 화면 정보 | 화면 이름, 화면 ID, 메뉴 위치/뎁스, 접근 권한 |
| 와이어프레임 | Figma 프레임 내 번호 콜아웃으로 UI 요소 표시 |
| 상세 설명 | 번호별 액션, 이동 화면, 팝업 조건, 엠티/에러/상태 케이스 |

### 0-2. 브랜드 에셋 반영 기준

`Brand_Assets/` 분석 결과, Stology의 시각 언어는 아래 기준으로 정리한다.

| 항목 | 반영 내용 |
|---|---|
| Primary | `#5B5CF6` |
| Sub | `#C7C9FF` |
| Text | `#1F2430` |
| Background | `#F5F6FF` |
| 캐릭터 | `Stolo`: 지식 네비게이터, 학습 도우미 |
| 그래픽 모티프 | 노드, 연결선, 지식 그래프, 성장/검증 |
| 키워드 | 연결, 성장, 신뢰, 협업, 기록 |

> 와이어프레임은 Mid-Fi 단계이므로 최종 Hi-Fi 이미지 중심 디자인이 아니라, 브랜드 색상/모티프/캐릭터 사용 위치를 화면 설계 수준에서 정의한다.

## 1. 인증 및 진입

### 1-1. 카카오 로그인

| 항목 | 내용 |
|---|---|
| 화면 이름 | 로그인 |
| 화면 ID | `LGN001` |
| 뎁스 | 비로그인 → 로그인 |
| 권한 | 비로그인 |
| Figma 프레임 | `04주차 - 와이어프레임 / LGN001 로그인` |

#### 와이어프레임 번호

| 번호 | UI 요소 | 상세 설명 |
|---|---|---|
| 1 | 카카오 로그인 버튼 | 클릭 시 카카오 OAuth 페이지로 이동한다. 인증 성공 시 기본적으로 홈 `HOM001`로 이동한다. |
| 2 | 초대 링크 진입 안내 | 초대 링크로 들어온 비로그인 사용자는 로그인 성공 후 해당 스터디 페이지 `STD001`로 자동 이동한다. |

#### 케이스별 화면

| 케이스 | 조건 | 결과 화면/상태 |
|---|---|---|
| 기본 로그인 | 일반 경로에서 카카오 로그인 성공 | 홈 `HOM001` |
| 초대 링크 로그인 | 초대 URL로 진입 후 로그인 성공 | 초대받은 스터디의 지식 구조 탭 `STD001` |
| 인증 실패 | 카카오 인증 실패 | 카카오 기본 에러 또는 로그인 화면 토스트 |
| 비로그인 접근 | 인증 필요한 URL 직접 접근 | `LGN001`로 리다이렉트 |

## 2. 홈 및 스터디 생성

### 2-1. 홈 화면 조회

| 항목 | 내용 |
|---|---|
| 화면 이름 | 홈 |
| 화면 ID | `HOM001` |
| 뎁스 | 로그인 → 홈 |
| 권한 | 로그인 사용자 |
| Figma 프레임 | `04주차 - 와이어프레임 / HOM001 홈/스터디 생성` |

#### 와이어프레임 번호

| 번호 | UI 요소 | 상세 설명 |
|---|---|---|
| 1 | 새 스터디 생성 카드 | 클릭 시 스터디 생성 모달을 호출한다. 진행 중 스터디가 없어도 항상 노출한다. |
| 2 | 활동 로그 | 최신순으로 개인 활동과 팀원 활동을 노출한다. 항목 클릭 시 해당 스터디/자료/질문 위치로 이동한다. |
| 3 | 스터디 생성 모달 | 필수값 입력 후 `생성하기` 클릭 시 초대 링크 모달로 자동 전환한다. |

#### 케이스별 화면

| 케이스 | 조건 | 결과 화면/상태 |
|---|---|---|
| 참여 스터디 있음 | 진행 중 스터디 1개 이상 | 스터디 카드 가로 목록 + 새 스터디 생성 카드 |
| 참여 스터디 없음 | 진행 중 스터디 0개 | 빈 안내 문구 + 새 스터디 생성 카드 |
| 종료 스터디 있음 | 종료된 스터디 존재 | 사이드바 `지난 활동` 아래에 표시 |
| 활동 없음 | 개인/팀원 활동 로그 0개 | 각 영역에 엠티 문구 표시 |

### 2-2. 스터디 생성

| 항목 | 내용 |
|---|---|
| 화면 이름 | 스터디 생성 모달 |
| 화면 ID | `HOM001-0100` |
| 뎁스 | 홈 → 새 스터디 생성 |
| 권한 | 로그인 사용자 |

#### 세부 기능

| 기능 | UI/입력 | 정책 |
|---|---|---|
| 스터디 이름 입력 | 필수, 최대 20자 | 중복 허용 |
| 온톨로지 템플릿 선택 | 필수, 드롭다운 | 템플릿 이름만 노출 |
| 시작일 입력 | 필수, `YYYY-MM-DD` | 과거 날짜 허용 |
| 설명 입력 | 선택, 최대 200자 | 스터디 카드/상세 설명에 활용 가능 |
| 생성하기 | 필수값 충족 시 활성 | 생성 직후 초대 링크 모달 호출 |

#### 케이스별 화면

| 케이스 | 조건 | 결과 화면/상태 |
|---|---|---|
| 필수값 미입력 | 이름/템플릿/시작일 중 누락 | `생성하기` 버튼 비활성 |
| 생성 성공 | 모든 필수값 충족 | 초대 링크 모달 `HOM001-0200` |
| 생성 실패 | 서버 오류/검증 실패 | 모달 유지 + 토스트 에러 |

### 2-3. 초대 링크 공유

| 항목 | 내용 |
|---|---|
| 화면 이름 | 초대 링크 모달 |
| 화면 ID | `HOM001-0200`, `MGT001-0100` |
| 뎁스 | 홈 → 스터디 생성 → 초대 링크 / 스터디 → 설정 → 초대 링크 |
| 권한 | 스터디장 |

| 기능 | 액션 | 결과 |
|---|---|---|
| 초대 링크 복사 | `복사하기` 클릭 | 클립보드 복사 + “링크가 복사되었습니다” 토스트 |
| 모달 닫기 | `닫기` 또는 X 클릭 | 생성 직후 진입이면 홈, 설정 진입이면 스터디 화면 복귀 |

## 4. 지식 구조

### 4-1. 그래프 탐색

| 항목 | 내용 |
|---|---|
| 화면 이름 | 지식 구조 탭 |
| 화면 ID | `STD001` |
| 뎁스 | 홈 → 스터디 → 지식 구조 |
| 권한 | 스터디 멤버 |
| Figma 프레임 | `04주차 - 와이어프레임 / STD001 지식 구조` |

#### 와이어프레임 번호

| 번호 | UI 요소 | 상세 설명 |
|---|---|---|

| 2 | 지식 그래프 캔버스 | 노드와 관계선을 표시한다. 활성 노드는 브랜드 Primary, 보조/비활성 노드는 Sub/Neutral로 표현한다. |
| 3 | 노드 상세 패널 | 노드 클릭 시 우측 패널이 열린다. 노드 정의, 관련 자료, 관계별 연결 노드를 표시한다. |

#### 케이스별 화면

| 케이스 | 조건 | 결과 화면/상태 |
|---|---|---|
| 활성 노드 클릭 | 자료가 연결된 노드 클릭 | 상세 패널에 관련 자료 목록 노출 |
| 비활성 노드 클릭 | 아직 활성화되지 않은 노드 클릭 | 정의와 연결 노드만 표시, 자료 0개 상태 |
| 빈 공간 클릭 | 그래프 배경 클릭 | 상세 패널 닫힘, 강조 해제 |
| 원본 자료 보기 | 관련 자료 클릭 | 원본 자료 조회 팝업 호출 |

## 5. 자료 업로드 및 AI 추출

### 5-1. 자료 입력

| 항목 | 내용 |
|---|---|
| 화면 이름 | 자료 업로드 탭 |
| 화면 ID | `UPL001` |
| 뎁스 | 홈 → 스터디 → 자료 업로드 |
| 권한 | 스터디 멤버 |
| Figma 프레임 | `04주차 - 와이어프레임 / UPL001 자료 업로드` |

#### 와이어프레임 번호

| 번호 | UI 요소 | 상세 설명 |
|---|---|---|
| 1 | 입력 방식 선택 | `파일 업로드` 또는 `텍스트 직접 입력` 중 하나를 선택한다. |
| 2 | 파일 드롭존 | `.md` 파일만 허용한다. 드래그 앤 드롭 또는 파일 선택을 지원한다. |
| 3 | 필수 입력 필드 | 주차, 자료 제목은 필수다. 자료 설명은 선택이다. |
| 4 | 대기 중인 자료 리스트 | 추출/검토 상태별 자료를 정렬해 보여준다. |

#### 케이스별 화면

| 케이스 | 조건 | 결과 화면/상태 |
|---|---|---|
| 파일 업로드 선택 | 파일 업로드 탭 활성 | 드롭존, 파일 선택 버튼 표시 |
| 텍스트 직접 입력 선택 | 텍스트 입력 탭 활성 | 여러 줄 텍스트 입력 영역 표시 |
| 등록 성공 | 필수값 충족 후 등록 | AI 추출 시작, “업로드 완료, 잠시 후 검토 가능” 토스트 |
| 필수값 미입력 | 주차/제목 누락 | 등록 버튼 비활성 |
| 업로드 실패 | 파일 형식/서버 오류 | 토스트 에러, 입력값 유지 |

### 5-2. 대기 자료 상태 처리

| 상태 | 의미 | 클릭 동작 |
|---|---|---|
| 추출 중 | AI 추출 진행 중 | 클릭 반응 없음 |
| 추출 실패 | AI 추출 실패 | 클릭 반응 없음, 1일 후 자동 삭제 |
| 검토 필요 | AI 후보 추출 완료 | 검토 화면 `REV001`로 이동 |
| 수정하기 | 본인 검토 완료, 결과 미확정 | 검토 화면 `REV001` 재진입 |

## 6. AI 추출 결과 검토

### 6-1. 노드 후보 검토

| 항목 | 내용 |
|---|---|
| 화면 이름 | AI 추출 결과 검토 |
| 화면 ID | `REV001` |
| 뎁스 | 홈 → 스터디 → 자료 업로드 → 검토 |
| 권한 | 스터디 멤버 |
| Figma 프레임 | `04주차 - 와이어프레임 / REV001 AI 후보 검토` |

#### 와이어프레임 번호

| 번호 | UI 요소 | 상세 설명 |
|---|---|---|
| 1 | 검토 진행률 배너 | 본인 기준 검토 완료 수를 `6/10 검토 완료` 형태로 표시한다. |
| 2 | 노드 후보 카드 | 후보 노드명, Evidence, 승인/반려 현황, 본인 액션 버튼을 표시한다. |
| 3 | 일괄 액션/검토 마치기 | 전체 승인, 선택 일괄 처리, 검토 마치기 버튼을 제공한다. |

#### 케이스별 화면

| 케이스 | 조건 | 결과 화면/상태 |
|---|---|---|
| 후보 승인 | `승인` 클릭 | 본인 검토 상태 승인으로 변경 |
| 후보 반려 | `반려` 클릭 | 본인 검토 상태 반려로 변경 |
| 미검토 후보 있음 | 모든 후보에 액션하지 않음 | `검토 마치기` 비활성 |
| 검토 제출 | 모든 후보 액션 완료 후 제출 | 자료 업로드 탭으로 복귀 |
| 승인 인원 충족 | 설정 인원 수 이상 승인 | 노드 활성화, 대기 리스트에서 제거 |
| 반려 발생 | 1명 이상 반려 | 노드 활성화 안 됨 |
| 후보 0개 | AI 추출 결과 없음 | “스터디와 관련 있는 주제를 업로드해주세요” 엠티뷰 |
